# التحقق من جاهزية البيئة والمخطط

## قبل الدمج أو النشر

1. **ترحيلات Supabase**  
   طبّق الترحيلات على المشروع المستهدف (`supabase db push` أو لوحة SQL)، ثم تأكد أن الجداول والدوال موجودة (مثلاً عدم ظهور خطأ «table not in schema cache»).

2. **أنواع TypeScript**  
   بعد تطبيق الترحيلات، ولّد الأنواع من نفس المرجع:
   ```bash
   npm run gen:types
   ```
   (يتطلب Supabase CLI وقاعدة محلية أو ربطاً بالمشروع.)

3. **التحقق الآلي للواجهة**
   ```bash
   cd frontend && npm run verify
   ```
   ينفّذ: `lint` → `test` → `build`.

## ملاحظة

البيئات (تطوير / staging / إنتاج) يجب أن تشترك في **نفس ترتيب الترحيلات** لتفادي اختلاف السلوك أو الأنواع.

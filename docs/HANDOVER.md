# تسليم المشروع — دليل المستلم (Handover)

> **الغرض:** أن يستطيع أي مطور جديد تشغيل المشروع، فهم أين يضع التعديلات، وصيانة الكود دون ضياع.  
> **الجمهور:** مالك المنتج + مطور يستلم العمل لاحقًا.

---

## 1) ماذا يفعل التطبيق؟ (جملة واحدة)

**مهمة التوصيل (Muhimmat)** — تطبيق ويب لإدارة المناديب والحضور والطلبات والرواتب والسلف والمركبات والتنبيهات، مع مصادقة وصلاحيات، مبني على **React + TypeScript + Vite** وخلفية **Supabase** (Postgres + Auth + RLS).

---

## 2) تشغيل المشروع (بدون افتراضات خاطئة)

| الموقع | الأمر | ملاحظة |
|--------|--------|--------|
| **جذر المستودع** (`muhimmat-1/`) | `npm install` ثم `npm run dev` | يشغّل الواجهة عبر `npm --prefix frontend` |
| **مجلد الواجهة** (`frontend/`) | `npm install` ثم `npm run dev` | يشغّل Vite مباشرة |

- **المنفذ الافتراضي للتطوير:** `5000` (انظر `frontend/vite.config.ts`).
- **البناء للإنتاج:** من الجذر `npm run build` أو من `frontend/` → `npm run build`.
- **الاختبارات:** `npm run test` من الجذر أو `frontend/`.

### متغيرات البيئة

- يُفترض وجود ملفات بيئة مثل `.env` / `.env.local` (غير مرفوعة في Git).  
- عميل Supabase: `frontend/services/supabase/client.ts` (مع `VITE_*` حسب إعداد المشروع).

---

## 3) أين الكود؟ — الهيكل الفعلي (ليس `src/` القديم)

الكود المصدري للتطبيق تحت **`frontend/`**:

| المسار | الدور |
|--------|--------|
| `frontend/app/` | نقطة الدخول، الراوتر، المزودين (Auth, Theme, i18n…)، `App.tsx` |
| `frontend/modules/` | صفحات كبيرة وميزات حسب المسار (مثل `modules/pages/`) |
| `frontend/shared/` | مكونات مشتركة، hooks، `lib`، واجهات UI المساعدة |
| `frontend/services/` | **طبقة الوصول للبيانات** — استدعاءات Supabase/RPC، أخطاء موحّدة |
| `frontend/services/supabase/` | عميل Supabase + أنواع DB |

> **قاعدة ذهبية للصيانة:** تعديلات جلب البيانات أو الجداول تمر عبر **`services/`** ثم تُستدعى من **hooks** أو **صفحات**، وليس نسخ استعلامات عشوائية داخل الصفحة.

---

## 4) ملفات “نقطة الدخول” للمراجعة السريعة

| الملف | لماذا يهم |
|-------|-----------|
| `frontend/app/App.tsx` | React Router، `QueryClientProvider`، `Toaster`، إعدادات Query الافتراضية |
| `frontend/app/providers/AuthContext.tsx` | الجلسة، المستخدم، الدور، تسجيل الخروج |
| `frontend/services/serviceError.ts` | `ServiceError`، `handleSupabaseError` — أسلوب الأخطاء في الخدمات |
| `frontend/shared/hooks/useAuthQueryGate.ts` | تفعيل/تعطيل الاستعلامات حسب الجلسة والمستخدم |

---

## 6) نطاق خارج المستودع (مذكور للتسليم فقط)

- **النشر:** غالبًا Vercel (إعدادات في `vercel.json` / `frontend/vercel.json` حسب التهيئة).  
- **Supabase:** لوحات التحكم، الـ migrations، وسياسات RLS — تُدار خارج هذا الكود ويجب توثيقها للمستلم.

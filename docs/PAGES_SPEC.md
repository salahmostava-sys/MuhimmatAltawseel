# وصف صفحات النظام — Muhimmat AlTawseel

> هذا الملف يصف كل صفحة في النظام: الغرض، المكونات، البيانات، والتفاعلات.
> مناسب لإعادة البناء بأي أداة (Google Stitch, v0, أو غيرها).

---

## المعلومات العامة

- **الاتجاه**: RTL (عربي)
- **التصميم**: Tailwind CSS + shadcn/ui (Radix)
- **الألوان**: نظام Light/Dark يتبع CSS variables
- **الخط**: النظام الافتراضي (system font stack)
- **الأيقونات**: Lucide React
- **الرسوم البيانية**: Recharts
- **الـ Layout**: Sidebar (قابل للطي) + Header مع avatar + محتوى رئيسي

## 1. تسجيل الدخول (`/login`)

**الغرض**: مصادقة المستخدم

**التصميم**: صفحة مقسومة نصفين:
- **الجزء الأيسر (55%)**: خلفية gradient أزرق + لوجو الشركة كـ watermark + اسم المشروع + 4 ميزات (أيقونة + عنوان + وصف)
- **الجزء الأيمن (45%)**: فورم تسجيل دخول

**المكونات**:
- حقل البريد الإلكتروني (مع أيقونة Mail)
- حقل كلمة المرور (مع زر إظهار/إخفاء + تحذير Caps Lock)
- Checkbox "تذكرني"
- زر تسجيل الدخول (gradient أزرق)
- زر تبديل Dark/Light mode
- رسالة خطأ عند فشل الدخول

**البيانات**: `auth.signIn(email, password)` → Supabase Auth

## 2. لوحة التحكم (`/`)

**الغرض**: نظرة عامة شاملة على أداء الشركة

**التبويبات** (4 tabs):

### Tab 1: النظرة العامة
- **6 بطاقات KPI**: إجمالي الطلبات، متوسط/مندوب، تحقيق الهدف، أفضل مندوب اليوم، متوسط التقييم، التنبيهات
- **تحليلات ذكية AI**: insights + توصيات مُولّدة من البيانات
- **مقارنة شهرية**: الشهر الحالي vs السابق (طلبات + أيام عمل)
- **مقارنة أسبوعية**: هذا الأسبوع vs الماضي
- **أداء المنصات**: بطاقة لكل منصة (طلبات، مناديب، هدف، نمو)
- **توزيع حسب المدينة**: مكة / جدة
- **تنبيهات ذكية**: مناديب متراجعين أو مختفين
- **جدول تفصيلي**: كل المناديب مع أدائهم

### Tab 2: التحليلات
- **رسم بياني شهري** (BarChart): اتجاه الطلبات آخر 6 أشهر
- **رسم بياني يومي** (LineChart): طلبات اليوم بيوم
- **مقارنة المنصات**: نمو كل منصة
- **توزيع الأداء**: ممتاز / متوسط / ضعيف (progress bars)

### Tab 3: التصنيف
- **4 أعمدة تصنيف**: Top 10، Worst 10، Most Improved، Most Declined
- كل مندوب: اسم، طلبات، نمو %، اتجاه

### Tab 4: المنصات
- **بطاقة لكل منصة**: طلبات، مناديب، progress bar تحقيق الهدف، نمو
- **رسم بياني** (BarChart): مقارنة الشهر الحالي بالسابق لكل منصة
- **جدول مقارنة** قابل للترتيب: 8 أعمدة

**البيانات**: `performance_dashboard_rpc(month_year)` → PostgreSQL function

**الجداول**: `employees`, `employee_apps`, `apps`, `salary_schemes`

## 4. الحضور والانصراف (`/attendance`)

**الغرض**: تسجيل الحضور اليومي + السجل الشهري

**التبويبات** (2 tabs):

### Tab 1: التسجيل اليومي
- **تقويم** لاختيار التاريخ
- **قائمة موظفين** مع dropdown لكل واحد: حاضر/غائب/إجازة/مريض/متأخر
- **حالات مخصصة** يمكن إضافتها
- **ألوان**: أخضر=حاضر، أحمر=غائب، أصفر=إجازة، أزرق=مريض، برتقالي=متأخر

### Tab 2: السجل الشهري
- **جدول**: الموظفين × أيام الشهر
- ملخص لكل موظف (أيام حضور، غياب، إجازات)

**الجداول**: `attendance`, `employees`, `apps`, `employee_apps`, `attendance_status_configs`

## 5. الطلبات (`/orders`)

**الغرض**: تسجيل الطلبات اليومية لكل مندوب على كل منصة

**التبويبات** (3 tabs):

### Tab 1: جدول الطلبات (Spreadsheet)
- **شبكة**: المناديب × أيام الشهر
- خلية = عدد الطلبات لمندوب في يوم معين على منصة معينة
- **فلترة بالمنصة** (tabs علوية لكل منصة)
- **إجماليات**: لكل مندوب (صف) + لكل يوم (عمود)
- **استيراد Excel** + تصدير + طباعة

### Tab 2: ملخص الشهر
- بطاقات إحصائية: إجمالي الطلبات، المناديب، الهدف
- تفاصيل لكل منصة مع حقل هدف قابل للتعديل

### Tab 3: الدوام
- **شبكة**: المناديب × أيام الشهر
- **3 حالات**: حاضر (أخضر) / غائب (أحمر) / فاضي (نقطة رمادية)
- **dropdown** عند الضغط على الخلية
- **المجموع**: أيام حضور / إجمالي أيام الشهر
- يُحتسب في الرواتب (حاضر × سعر اليوم من الإسكيما)

**الجداول**: `daily_orders`, `daily_shifts`, `apps`, `employees`, `employee_apps`, `app_targets`, `locked_months`

## 6. الرواتب (`/salaries`)

**الغرض**: حساب وإدارة رواتب الموظفين الشهرية

**المكونات**:
- **جدول رواتب ضخم**: كل موظف في صف مع أعمدة لكل منصة
- **أعمدة**: اسم الموظف، أيام العمل، طلبات/دوام لكل منصة، الراتب لكل منصة، الحوافز، الخصومات، السلف، صافي الراتب
- **ألوان المنصات**: كل منصة بلون brand الخاص بها
- **تعديل inline**: حوافز، خصومات يدوية
- **حالات**: مسودة / معتمد
- **كشف راتب**: PDF لكل موظف
- **تصدير**: Excel + PDF جماعي
- **محرك الرواتب**: يحسب تلقائياً من الطلبات + الدوام + الإسكيما

**الجداول**: `salary_records`, `daily_orders`, `advance_installments`, `employees`, `external_deductions`
**RPCs**: `preview_salary_for_month`, `calculate_employee_salary`
**Edge Function**: `salary-engine`

## 8. المصاريف والإيرادات (`/finance`)

**الغرض**: تتبع الميزانية الشهرية

**التصميم**: عمودين جنب بعض:

**العمود الأيسر — الإيرادات 💰**:
- جدول: التصنيف، الوصف، المبلغ، التاريخ، حذف
- **صف إضافة inline** في الأسفل: dropdown تصنيف + حقل مبلغ + تاريخ + زر إضافة
- تصنيفات: إيرادات توصيل، إيرادات منصات، عقود شركات، إيرادات أخرى

**العمود الأيمن — المصاريف 💸**:
- نفس الهيكل
- تصنيفات: رواتب، إيجار، وقود، صيانة مركبات، تأمين، اتصالات، مستلزمات، مصاريف إدارية، أخرى
- **الرواتب تلقائية** (🔒): زر "مزامنة الرواتب" يجلب إجمالي الرواتب المعتمدة

**3 بطاقات أعلى**: إجمالي الإيرادات (أخضر) + إجمالي المصاريف (أحمر) + الفرق/الرصيد

**الجداول**: `finance_transactions`, `salary_records`

## 9. استهلاك المناديب (`/fuel`)

**الغرض**: تتبع الكيلومترات وتكلفة البنزين

**3 عروض**:

### عرض شهري
- **جدول**: المندوب، أيام مسجّلة، كيلومترات، بنزين، تكلفة/كم، الدباب، الطلبات، بنزين/طلب
- **4 بطاقات**: إجمالي كم، بنزين، متوسط تكلفة/كم، الطلبات

### عرض يومي
- **قائمة مناديب** قابلة للتوسيع (accordion)
- كل مندوب: سجلات يومية (تاريخ، كم، بنزين، ملاحظات) + تعديل/حذف
- **صف إضافة** inline في الأسفل

### جدول بيانات (Spreadsheet)
- **شبكة**: المناديب × أيام الشهر
- **3 قيم في كل خلية**: 📦 طلبات (أزرق) + 🛣️ مسافة (أخضر) + ⛽ بنزين (أصفر)
- **هيدر مزدوج**: يوم الأسبوع + رقم اليوم، الجمعة بلون وردي
- **إجماليات**: لكل صف + لكل عمود
- **6 بطاقات إحصائية**: مناديب بها بيانات، طلبات، مسافة، بنزين، خلايا مسجلة، نسبة تغطية
- **Popover** عند الضغط: تفاصيل اليوم الكاملة

**فلترة بالمنصة** (tabs): الكل / كل منصة لوحدها

**الجداول**: `vehicle_mileage_daily`, `daily_orders`, `employees`, `apps`, `employee_apps`, `vehicle_assignments`, `vehicle_mileage`

## 10. التطبيقات/المنصات (`/apps`)

**الغرض**: إدارة منصات التوصيل (أمازون، هنقرستيشن، جاهز، إلخ)

**المكونات**:
- **بطاقات** لكل منصة: اسم، لون، عدد المناديب، نوع العمل (طلبات/دوام/مختلط)
- **إضافة/تعديل منصة** (modal): اسم، لون brand، نوع العمل
- **حذف**: أرشفة أو حذف نهائي (مع عرض البيانات المرتبطة)
- **لوحة الموظفين**: عند الضغط على منصة تظهر قائمة المناديب المربوطين

**إعدادات المنصات** (`/apps/settings`):
- إسكيمات الرواتب (شرائح سعرية) لكل منصة
- قواعد التسعير
- أهداف شهرية

**الجداول**: `apps`, `employee_apps`, `daily_orders`, `daily_shifts`, `pricing_rules`, `app_targets`, `app_hybrid_rules`

## 18. إعدادات النظام (`/settings`)

# ✅ نظام الصيانة وقطع الغيار - ملخص شامل

## 📋 نظرة عامة

نظام متكامل لإدارة صيانة المركبات وتتبع قطع الغيار مع **تحديث تلقائي للمخزون** وحسابات دقيقة.

---

## 🗄️ البنية التقنية

### الجداول الثلاثة الرئيسية:

| الجدول | الوظيفة | الحقول الرئيسية |
|--------|---------|-----------------|
| **spare_parts** | مخزون قطع الغيار | name_ar, stock_quantity, min_stock_alert, unit_cost |
| **maintenance_logs** | سجلات الصيانة | vehicle_id, employee_id, maintenance_date, total_cost |
| **maintenance_parts** | ربط القطع بالصيانة | maintenance_log_id, part_id, quantity_used, cost_at_time |

---

## ⚙️ الآليات التلقائية (4 Triggers)

### 1️⃣ تعبئة السائق تلقائياً
```
عند إضافة صيانة → يتم البحث عن السائق الحالي للمركبة → يتم تعبئة employee_id
```

### 2️⃣ خصم المخزون تلقائياً
```
عند إضافة قطعة للصيانة → يتم خصم الكمية من المخزون → يتم رفض العملية إذا كان المخزون غير كافٍ
```

### 3️⃣ استرجاع المخزون عند الحذف
```
عند حذف سجل صيانة → يتم إرجاع الكميات إلى المخزون تلقائياً
```

### 4️⃣ حساب التكلفة الإجمالية
```
عند إضافة/تعديل/حذف قطع → يتم حساب total_cost = SUM(quantity × cost_at_time)
```

---

## 🎯 سير العمل الكامل

### مثال عملي: صيانة دورية

```
1. المستخدم يفتح صفحة الصيانة
2. يضغط "إضافة صيانة"
3. يختار المركبة: "أ ب ج 1234"
   ✅ يظهر اسم السائق الحالي تلقائياً: "محمد أحمد"
4. يختار نوع الصيانة: "غيار زيت"
5. يدخل قراءة العداد: 50000 كم
6. يضيف القطع:
   - زيت محرك: 4 لتر × 25 ريال = 100 ريال
   - فلتر زيت: 1 قطعة × 30 ريال = 30 ريال
   - فلتر هواء: 1 قطعة × 20 ريال = 20 ريال
7. يضغط "حفظ الصيانة ✓"

✅ يحدث تلقائياً:
   - يتم إنشاء سجل في maintenance_logs
   - يتم ربط employee_id بـ "محمد أحمد"
   - يتم إضافة 3 سجلات في maintenance_parts
   - يتم خصم:
     * 4 لتر من زيت المحرك (100 → 96)
     * 1 قطعة من فلتر الزيت (50 → 49)
     * 1 قطعة من فلتر الهواء (30 → 29)
   - يتم حساب total_cost = 150 ريال
   - يتم تحديث updated_at لجميع الجداول
```

## 🛡️ الحماية والتحقق

### 1. منع المخزون السالب
- ✅ يتم التحقق في الواجهة قبل الحفظ
- ✅ يتم التحقق في قاعدة البيانات (Trigger)
- ✅ رسالة خطأ واضحة: "المخزون غير كافٍ للقطعة المطلوبة"

### 2. منع حذف قطع مستخدمة
- ✅ يتم التحقق من الاستخدام قبل الحذف
- ✅ رسالة خطأ: "لا يمكن حذف القطعة لأنها مستخدمة في سجلات صيانة"

### 3. سلامة البيانات
- ✅ جميع العمليات تتم داخل Transactions
- ✅ Foreign Keys تضمن الربط الصحيح
- ✅ Cascade Delete لحذف القطع عند حذف السجل

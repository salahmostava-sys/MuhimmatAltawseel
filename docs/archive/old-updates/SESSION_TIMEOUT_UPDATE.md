## 📋 الجدول الزمني

| الحدث | الوقت | الوصف |
|-------|-------|-------|
| بداية الجلسة | 0:00 | المستخدم يسجل الدخول |
| نشاط المستخدم | 0:00 - 9:00 | أي حركة تعيد تشغيل العداد |
| عدم النشاط | 9:00 | لا توجد حركة لمدة 9 دقائق |
| **التحذير** | **9:00** | **⚠️ رسالة: ستنتهي جلستك خلال دقيقة واحدة** |
| عرض التحذير | 9:00 - 9:58 | الرسالة تظهر لمدة 58 ثانية |
| **تسجيل الخروج** | **10:00** | **تسجيل خروج تلقائي** |

## 🎯 السلوك

### سيناريو 1: المستخدم نشط
```
0:00 → تسجيل الدخول
2:00 → حركة الماوس → إعادة تشغيل العداد ✅
5:00 → ضغط على زر → إعادة تشغيل العداد ✅
8:00 → كتابة → إعادة تشغيل العداد ✅
... المستخدم يبقى مسجلاً
```

### سيناريو 2: المستخدم غير نشط
```
0:00 → تسجيل الدخول
9:00 → ⚠️ تحذير: ستنتهي جلستك خلال دقيقة واحدة
10:00 → 🚪 تسجيل خروج تلقائي
```

### سيناريو 3: المستخدم يستجيب للتحذير
```
0:00 → تسجيل الدخول
9:00 → ⚠️ تحذير: ستنتهي جلستك خلال دقيقة واحدة
9:30 → حركة الماوس → إلغاء التحذير ✅ + إعادة تشغيل العداد
... المستخدم يبقى مسجلاً
```

### سيناريو 4: تبويبات متعددة
```
Tab 1: 0:00 → تسجيل الدخول
Tab 2: 0:00 → تسجيل الدخول (نفس الحساب)

Tab 1: 9:00 → ⚠️ تحذير
Tab 2: 9:00 → ⚠️ تحذير

Tab 1: 9:30 → حركة الماوس
Tab 2: 9:30 → إلغاء التحذير تلقائياً ✅ (مزامنة)

... كلا التبويبين يبقيان نشطين
```

```bash
# 4. يظهر التحذير في كلا التبويبين
# 5. حرك الماوس في التبويب الأول
# 6. يجب أن يختفي التحذير في كلا التبويبين (مزامنة)
```

### للمستخدمين
- ✅ **أقل إزعاجاً:** لن يتم تسجيل الخروج بشكل متكرر
- ✅ **وقت أطول للاستجابة:** دقيقة كاملة للرد على التحذير
